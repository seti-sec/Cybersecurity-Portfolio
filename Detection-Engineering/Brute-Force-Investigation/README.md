# Brute Force Investigation

## Objective
Detect and investigate possible brute force attacks against user accounts by identifying multiple failed authentication attempts within a short period of time.
## Incident Scenario
A user reports multiple failed login notifications. 
The SOC team investigates authentication logs and discovers that an external IP address generated hundreds of failed login attempts against multiple accounts.
The attacker is attempting to guess valid credentials and gain unauthorized access.
## Environment
- Windows Server 2022
- Windows Security Logs
- Splunk Enterprise
- Universal Forwarder
- Sysmon
## Data Sources / Log Sources
The investigation uses:
- Windows Security Event Logs
- Authentication Logs
- VPN Logs
- Firewall Logs
Important Event IDs:
4625 - Failed Logon
4624 - Successful Logon
## MITRE ATT&CK Mapping
T1110 - Brute Force
Sub-techniques:
T1110.001 - Password Guessing
T1110.003 - Password Spraying
## Detection Logic
Trigger an alert when:
- More than 10 failed login attempts
- From the same source IP
- Against one or multiple accounts
- Within 5 minutes
## Query

### SPL (Splunk)
index=wineventlog EventCode=4625
| bin _time span=5m
| stats count as failed_attempts values(Account_Name) as usernames by src_ip,_time
| where failed_attempts >= 10

The SPL detection rule is available here:
[brute_force_detection.spl](queries/brute_force_detection.spl)
### KQL (Microsoft Sentinel)
SecurityEvent
| where EventID == 4625
| summarize FailedAttempts=count(),
Users=make_set(Account),
IPs=make_set(IpAddress)
by IpAddress, bin(TimeGenerated, 5m)
| where FailedAttempts >= 10
## Investigation Steps
1. Identify source IP address.
2. Check targeted usernames.
3. Review successful logins after failed attempts.
4. Check IP reputation.
5. Determine if the account was compromised.
## Timeline
10:00 - 50 failed login attempts detected
10:05 - Alert generated by SIEM
10:10 - Successful login detected
10:15 - Account disabled
## Findings
The investigation identified 250 failed login attempts from external IP 185.x.x.x targeting multiple accounts.
One successful login occurred after the brute force activity.
## False Positives
* User forgot password
* VPN reconnect attempts
* Service accounts with expired passwords
* Misconfigured applications
## Response / Mitigation
* Block malicious IP
* Reset compromised password
* Enable MFA
* Disable compromised account
* Review affected systems
## Lessons Learned
* Enable MFA
* Monitor authentication anomalies
* Implement account lockout policies
* Improve detection rules
