# Kerberoasting Detection

## Objective

Detect and investigate possible Kerberoasting attacks by identifying suspicious Kerberos Service Ticket requests that may indicate credential access attempts against service accounts.

## Incident Scenario

A SOC analyst receives an alert indicating unusual Kerberos authentication activity inside the domain environment.

During investigation, the security team discovers that a domain user account requested multiple Service Tickets (TGS) for different Service Principal Names (SPNs) within a short period of time.

The attacker is attempting to obtain Kerberos service tickets and perform offline password cracking against service account credentials.

Important Event IDs:

4769 - A Kerberos service ticket was requested

## MITRE ATT&CK Mapping

T1558 - Steal or Forge Kerberos Tickets

Sub-techniques:

T1558.003 - Kerberoasting

## Detection Logic

Trigger an alert when:

- Multiple Kerberos Service Ticket requests are generated by the same user
- Requests target multiple Service Principal Names (SPNs)
- RC4 encryption type is used (0x17)
- Activity occurs within a short time window
- The requesting account is not expected to request large numbers of service tickets

## Query

### SPL (Splunk)

```spl
index=wineventlog EventCode=4769
Ticket_Encryption_Type=0x17

### KQL (Microsoft Sentinel)

SecurityEvent
| where EventID == 4769
| where TicketEncryptionType == "0x17"
| summarize 
    TicketRequests=count(),
    SPNs=make_set(ServiceName)
    by Account, IpAddress, bin(TimeGenerated, 5m)

## Investigation Steps
1. Identify the account requesting Kerberos service tickets.
2. Review requested SPNs and affected service accounts.
3. Check encryption type used in ticket requests.
4. Verify if the user normally requests these services.
5. Investigate possible credential compromise.
6. Review endpoint activity associated with the account.

## Findings
The investigation identified multiple Kerberos service ticket requests from a domain user account targeting several service accounts.
The activity matched common Kerberoasting behavior due to the use of RC4 encryption and abnormal SPN request volume.

## False Positives
* Legitimate administrative tools requesting multiple service tickets
* Service accounts performing normal authentication activity
* Domain management scripts
* Enterprise applications using Kerberos authentication

## Response / Mitigation
* Reset compromised service account passwords
* Disable unnecessary SPNs
* Enforce strong service account passwords
* Disable RC4 where possible
* Enable monitoring for abnormal Kerberos activity
* Implement least privilege access

## Lessons Learned
* Monitor Kerberos authentication anomalies
* Use strong passwords for service accounts
* Reduce unnecessary service account privileges
* Improve Active Directory visibility
* Continuously tune detection rules
